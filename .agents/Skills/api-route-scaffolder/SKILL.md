# SKILL.md — API Route Scaffolder

This document outlines the required technical competencies and domain knowledge needed to design, scaffold, and implement the backend API routes for the Booktrovert MVP. In our architecture, the Supabase client handles standard CRUD operations directly from the frontend. Therefore, API routes are exclusively reserved for **server-side orchestration of external integrations** to keep API keys secure.

## 1. API Design & Architecture
Building intuitive, predictable, and scalable serverless endpoints for external proxies.

* **Serverless Proxies:** Designing lightweight endpoints dedicated to protecting external API keys (e.g., `GET /api/books/search`, `POST /api/recommendations/generate`).
* **Payload Structuring:** Designing clean JSON request and response contracts, ensuring the frontend receives exactly what it needs without over-fetching.
* **Idempotency & State Management:** Ensuring routes are reliable and handle race conditions gracefully.

## 2. Security & Authentication Implementation
Executing the strict rules defined in the `auth-flow.md` specification.

* **Session Validation:** Supabase handles sessions natively via the client. Backend API routes must simply validate the incoming Supabase JWT token before allowing the route to execute.
* **Vulnerability Mitigation:** Implementing rate limiting on sensitive, high-compute server-side routes (`/api/books/search`, `/api/recommendations/generate`) to prevent API abuse.
* **Sanitization:** Stripping all Personally Identifiable Information (PII) before returning data to the client, especially on any public share route wrappers.

## 3. Database Integration & Querying
Interacting with the Supabase database from the server-side environment.

* **Supabase Server Client:** Using the Supabase JS client in a server/serverless context to query and mutate data when an operation is too complex or sensitive for the frontend (e.g., saving recommendations generated by DeepSeek directly to the DB before returning to the client).
* **Relational Traversing:** Handling complex joins efficiently using Supabase's built-in PostgREST syntax (e.g., `supabase.from('userbooks').select('*, books(*)')`).

## 4. External Integrations Orchestration
Managing the flow of data between the backend and third-party services.

* **Books API Handling:** Querying external APIs (Open Library, Google Books), standardizing their inconsistent response shapes into Booktrovert's exact `Book` schema, and handling timeouts or 0-result fallbacks smoothly.
* **LLM Engine Connectivity:** Packaging the user's tag data, sending optimized prompts to the DeepSeek V3 API, enforcing JSON output, stripping markdown, and mapping it to the `Recommendation` table.

## 5. Validation & Error Handling
Ensuring the API never crashes and always returns actionable feedback to the client.

* **Schema Validation:** Using libraries (like Zod or Joi) to strictly validate incoming request bodies (e.g., ensuring manual book entries have a title, author, and at least one context tag before querying APIs).
* **Standardized Error Responses:** Returning consistent HTTP status codes and error messages (e.g., `400 Bad Request` for validation failures, `503 Service Unavailable` when the external Books API fails).
* **Logging & Monitoring:** Setting up basic server-side logging to track external API failure rates and DeepSeek latency.
