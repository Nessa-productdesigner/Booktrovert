## userbooks

This is the core join table. It stores shelf status and context tags per user per book.
Tags are never merged across users — each record is fully independent.

| Column | Type | Notes |
|---|---|---|
| userbook_id | UUID | Primary key |
| user_id | UUID | Foreign key → users |
| book_id | UUID | Foreign key → books |
| shelf | ENUM | to_read \| currently_reading \| read \| rereading \| did_not_finish |
| context_tags | JSON | All tag dimensions + selected values per user |
| added_at | TIMESTAMP | Auto |
| updated_at | TIMESTAMP | Auto-updates on every edit |

### context_tags JSON structure
```json
{
  "pacing": ["slow burn"],
  "emotional_tone": ["melancholic", "tense"],
  "writing_style": ["lyrical"],
  "structure": ["unreliable narrator"],
  "tropes": ["enemies to lovers", "morally grey protagonist"],
  "feeling_after_finishing": ["wrecked"]
}
```

## Rules

- Tags are stored per userbook record — never globally per book
- Same book logged by two different users stores two independent userbook records
- Did Not Finish shelf tags are stored as normal but flagged as negative signals
  by the recommendation engine — not by the schema
- Tag edits update the updated_at field and trigger a recommendation re-run
- Manual books have source = manual and may have null cover_url and synopsis
